# testing/

```
testing/
├── software_tests/        unit + API tests (see its README for what's actually been executed)
├── hardware_tests/        bench validation checklist for the physical build
└── performance_reports/   original baseline metrics + a log template for your own measurements
```

Start with `software_tests/README.md` for the code-level test status, and
`performance_reports/baseline_original_prototype.md` before quoting any performance number from this
project elsewhere (a resume, a report, an interview) — it explains exactly which numbers are measured,
on which version of the system, and which still need re-validation.
