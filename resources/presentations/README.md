# resources/presentations/

No slide deck is included in this revision of the repository.

If you're preparing a presentation from this project (e.g., for a college review, conference, or
interview portfolio walkthrough), a good structure to pull from the rest of this repo:

1. **Problem & motivation** — root `README.md` § Problem Statement / Motivation.
2. **System architecture** — `diagrams/System_Architecture.svg` + `diagrams/Block_Diagram.svg`.
3. **What changed from the original academic build** — root `README.md` § Architecture Corrections.
4. **Live demo or photos** — `images/`.
5. **Results** — root `README.md` § Performance Metrics (note the honest baseline-vs-target framing
   there before presenting any number as a finished result).
6. **Future work** — root `README.md` § Future Enhancements.

A `.pptx` version of this outline can be generated on request — ask whoever's helping you build this
repository (e.g., re-open this conversation with Claude and ask for a slide deck) rather than leaving a
stale, hand-maintained copy to go out of sync with the README.
